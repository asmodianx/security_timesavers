Blameless Post Mortem Guide

Stand-alone incident review worksheet for security and operational events. Works locally or from a web server. No CDN or external dependencies.

Use this to guide a blameless after action review, preserve structured data, export JSON, create a readable PDF, and generate an LLM-ready prompt package with framing and guard rails.
Local save uses browser storage on the current device/browser. JSON export is portable.

Blameless facilitation notes

  • Focus on what happened, why system conditions allowed it, and how to reduce future risk.
  • Avoid naming people as root cause. Capture process, tooling, detection, communication, and environmental factors.
  • Mark uncertainty explicitly. Distinguish facts, assumptions, open questions, and hypotheses.
  • For security incidents, include attacker path, containment, recovery, evidence confidence, data impact, and control gaps.
  • Do not paste secrets, passwords, private keys, access tokens, or sensitive personal data into the report.
  • Use the LLM export only after removing restricted data and reviewing prompt guard rails.
