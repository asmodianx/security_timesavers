Host / IP Enrichment Tool

Self-contained browser tool for passive host enrichment using DNS, reverse DNS, RDAP, and multiple JSON lookup tables. (This uses DNS so there is some data leakage possible here)

1) Input targets

Enter one IP, hostname, or URL per line.

2) Load lookup tables

Supported table types: cidr, ip, hostname, domain_suffix, regex

No lookup tables loaded.

Lookup Table JSON Format

Each lookup file must be a JSON object with a type and an entries array. The tool accepts multiple lookup files at once and applies all matching entries to each host.

Required top-level fields

  • name — Friendly name shown in the UI.
  • type — One of: cidr, ip, hostname, domain_suffix, regex.
  • entries — Array of lookup records.
  • description — Optional human-readable description.
  • defaultTags — Optional tags automatically added to every match from this table.

Required entry fields

  • value — The value to match (for example a CIDR, IP, hostname, suffix, or regex pattern).
  • label — Friendly label shown in results.
  • severity — Optional. Suggested values: info, low, medium, high, critical.
  • tags — Optional string array added when the entry matches.
  • meta — Optional object for arbitrary structured metadata.

Type behavior

  • cidr — Matches the resolved IP against a CIDR block like 10.0.0.0/8.
  • ip — Exact IP match like 8.8.8.8.
  • hostname — Exact hostname match like vpn.example.org.
  • domain_suffix — Hostname ends-with match like .example.org.
  • regex — JavaScript regular expression pattern matched against the normalized input, hostname, and IP.

Practical notes
  • Use one JSON file per logical source when possible (for example internal_networks.json, watchlist_ips.json, suspicious_domains.json).
  • The same host may match multiple tables and multiple entries. All hits are preserved.
  • For regex tables, use standard JavaScript regex syntax without surrounding slashes.
  • The meta object is preserved in exports and is good for owner, ticket, source feed, disposition, or notes.
  • If you want tags added automatically to every entry in the file, use defaultTags.
