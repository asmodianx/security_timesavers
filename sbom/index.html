Standards notes

What this app supports: CycloneDX JSON 1.6/1.7 and SPDX JSON 2.3 import, viewing, validation, normalization, and export. It focuses on the most common JSON-based SBOM workflows.
Why only these JSON standards?

CycloneDX and SPDX both have well-established JSON serializations. SWID is an important software-identification standard, but its formal serializations are XML and CBOR rather than common JSON. This app therefore treats JSON as the primary interchange target and alerts if a document appears to be neither CycloneDX nor SPDX.

Validation philosophy

Validation is intentionally pragmatic rather than exhaustive schema validation. The alert engine checks for key top-level fields, minimum expected component fields, duplicate identifiers, malformed timestamps, odd license/hash structures, and dependency references to missing components. That makes it useful offline even without bundling official schemas.

Builder model

Internally, the builder uses a simple normalized model: document metadata, one root component, child components, optional hashes/licenses/identifiers, and dependency edges. Exporters map this model into CycloneDX or SPDX JSON.
